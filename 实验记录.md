# DDPM InSAR 相位解缠 — 实验记录

## 项目概述

基于去噪扩散概率模型 (DDPM) 的 InSAR 相位解缠算法。输入为包裹相位和相干性图，输出为解缠相位。使用 `taskbook_merged` 数据集（含真实 LiCSAR 数据和合成数据），当前训练用 300 样本、验证用 80 样本。

### 项目文件结构

```
Easy_DDPM/
├── config.py              # 所有超参数配置
├── dataset.py             # 数据加载 (InSARDataset, make_dataloaders)
├── model.py               # UNet + DDPM/DDIM 模型
├── train.py               # 训练脚本 (DDPM / UNet 两种模式)
├── unet_regression.py     # UNet 回归基线 (无扩散)
├── baselines.py           # 经典解缠方法 (LS / QG)
├── evaluate.py            # 四方法统一评估 (5 维度指标)
├── inference.py           # 推理与可视化
├── benchmark_ddim.py      # DDIM 速度-质量扫描
├── 实验设计.md             # 实验路线图
├── 实验记录.md             # 本文件
└── output/                # 模型 checkpoints 与评估输出
```

### 数据流

```
原始数据: 512×512 .npz patches
  ├── wrapped phase       [-π, π]          → sin/cos 编码
  ├── coherence           [0, 1]
  ├── unwrapped phase     (ground truth)
  ├── coarse_unwrapped    (LS 基线)
  ├── mask                (0/1 有效像素)
  └── 元数据              (pair_id, difficulty_score, 分组标签等)

预处理:
  1. 中心裁剪 512×512 → 256×256
  2. 条件构建: [sin(wrapped), cos(wrapped), coherence] → [3, 256, 256]
  3. 目标: unwrapped phase → [1, 256, 256]
```

### 四方法对比架构

| 方法 | 类型 | 核心原理 | 训练 | 推理时间 |
|------|------|---------|:---:|:---:|
| **最小二乘法 (LS)** | 经典FFT | 求解泊松方程 Δφ=∇·(∇ψ_wrapped) | 否 | 17 ms |
| **质量引导法 (QG)** | 经典路径跟踪 | 从高相干种子像素逐点积分，优先队列 | 否 | 62 ms |
| **UNet 回归** | 深度学习 | 直接映射 3ch 条件→1ch 相位 | 是 | 20 ms |
| **DDPM (本文)** | 扩散模型 | 200步去噪 + DDIM-50采样 | 是 | 298 ms |

## 实验环境

- **Python**: 3.12.13
- **PyTorch**: 2.11.0+cu126
- **GPU**: CUDA 12.6, AMP fp16
- **数据集**: `taskbook_merged` (train=2725, val=564, test=550, 512×512 patches)
  - 真实数据: 6 个 LiCSAR 干涉对 (1525 训练 patches)
  - 合成数据: 4 种形变 × 3 噪声 × 3 梯度 × 3 相干性 (1200 训练 patches)
- **训练配置**: 300 样本, 80 epochs, batch=4, AdamW lr=1e-4, EMA=0.995
- **评估配置**: 80 验证样本, DDIM-50 推理

### 评估指标

| 指标 | 含义 | 方向 |
|------|------|:---:|
| 掩膜平均绝对误差 (Masked MAE) | 仅有效像素的 |预测-真值| 均值 | ↓ |
| 掩膜均方根误差 (Masked RMSE) | 仅有效像素的 RMSE | ↓ |
| 包裹域平均绝对误差 (Wrapped MAE) | 预测相位重新包裹后与原始包裹观测的偏差 | ↓ |
| 梯度平均绝对误差 (Gradient MAE) | 空间梯度 (x/y) 的 MAE | ↓ |
| 优于最小二乘比例 (WinRate) | 该方法的 MAE ≤ LS 的 MAE 的样本比例 | ↑ |

## 模型架构详解

### 1. DDPM 去噪 UNet

```
输入层:
  x_t [B, 1, 256, 256] + condition [B, 3, 256, 256] → concat → [B, 4, 256, 256]
  conv_in: Conv2d(4→64, k=3, p=1)

时间步嵌入:
  Sinusoidal(128 dim) → Linear(128→512) → SiLU → Linear(512→128)
  每个 ResBlock 中: t_emb → Linear(128→out_ch) → add to feature

编码器 (3 级, channel_mults=[1,2,4]):
  Level 0: 256×256, ch=64
    ResBlock(64→64) ×2 → 保存跳连
    Downsample: Conv2d(64→64, k=3, s=2) → 128×128
  Level 1: 128×128, ch=128
    ResBlock(64→128) ×2 → 保存跳连
    Downsample: Conv2d(128→128, k=3, s=2) → 64×64
  Level 2: 64×64, ch=256
    ResBlock(128→256) ×2 + Self-Attention@32 → 保存跳连

中间层 (bottleneck):
  ResBlock(256→256) → Self-Attention → ResBlock(256→256)

解码器 (3 级, 逆序):
  Level 0: 64×64, ch=256
    上采样: nearest ×2 → conv(256→256) → 128×128
    拼接跳连 → ResBlock(512→256) ×3
  Level 1: 128×128, ch=128
    上采样: nearest ×2 → conv(128→128) → 256×256
    拼接跳连 → ResBlock(256→128) ×3
  Level 2: 256×256, ch=64
    拼接跳连 → ResBlock(128→64) ×3

输出层:
  GroupNorm(32, 64) → SiLU → Conv2d(64→1, k=3, p=1)

参数量: 12,617,921
```

### 残差块 (ResBlock) 结构

```
输入 x [B, C_in, H, W] + t_emb [B, 128]

GroupNorm(32, C_in) → SiLU → Conv2d(C_in→C_out, k=3, p=1)
  + t_emb → Linear(128→C_out) → add (spatial broadcast)

GroupNorm(32, C_out) → SiLU → Dropout(0.1) → Conv2d(C_out→C_out, k=3, p=1)
  + skip: x → Conv1x1(C_in→C_out) if C_in≠C_out, else identity
```

### DDPM 扩散过程

```
前向 (训练):  q(x_t | x_0) = N(√ᾱ_t·x_0, (1-ᾱ_t)·I)
  采样 t ~ Uniform(0,199), ε ~ N(0,I)
  x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε
  loss = MSE(ε_θ(x_t, t, cond), ε)

噪声调度: 线性, β = 1e-4 → 0.02, T=200

反向 (推理 DDIM-50):
  时间步子序列: [199, 195, 191, ..., 3] (50 steps)
  x̂_0 = (x_t - √(1-ᾱ_t)·ε_θ) / √ᾱ_t
  x_{t-1} = √ᾱ_{prev}·x̂_0 + √(1-ᾱ_{prev})·ε_θ  (deterministic)
```

### 2. UNet 回归网络 (对比基线)

```
结构与 DDPM UNet 相同，区别:
  - 无时间步嵌入: ResBlock2D 仅 GroupNorm→SiLU→Conv→GroupNorm→SiLU→Conv+skip
  - 无 dropout (推理模式)
  - 输入: 3ch 条件 → 输出: 1ch 解缠相位
  - 损失: MSE + 0.5·L1

参数量: 11,866,945
```

### 3. 经典解缠基线

**最小二乘法 (LS)**:
```
1. 最近邻填充无效像素
2. 计算包裹梯度 gx, gy (mod 2π)
3. 计算散度 div = ∂gx + ∂gy
4. FFT 求解泊松方程: φ = IFFT(FFT(div) / denominator)
5. 中位数去除常数偏移
```

**质量引导法 (QG)**:
```
1. 以相干性为质量图
2. 从最高质量种子像素开始
3. 优先级队列扩展 (高质量边优先)
4. 逐像素包裹差分积分
5. 参考 LS 结果进行 gauge 对齐
```

## 实验 1：基线 DDPM 模型 (128×128)

### 配置
| 参数 | 值 |
|------|-----|
| 图像尺寸 | 128×128 (512→128 中心裁剪) |
| 条件通道 | 3 (sin(wrapped), cos(wrapped), coherence) |
| 噪声步数 | 200 (线性调度 β=1e-4→0.02) |
| UNet | 64 base, [1,2,4] mults, 2 resblocks, attn@16 |
| 训练样本 | 300 (随机种子42) |
| Epochs | 80 |
| Batch | 8 |
| 优化器 | AdamW lr=1e-4, CosineAnnealingLR |
| 采样 | DDPM 200步 |

### 结果
| 方法 | Masked MAE | Masked RMSE | Wrapped MAE | Grad MAE | WinRate |
|------|:---:|:---:|:---:|:---:|:---:|
| Least-Squares | 14.00 | 14.71 | 1.53 | 0.95 | — |
| Quality-Guided | 13.86 | 14.02 | 0.00 | 0.77 | 45% |
| UNet Reg. | 7.49 | 7.79 | 1.80 | 0.62 | 91% |
| **DDPM (Ours)** | **8.33** | **8.71** | **1.69** | **0.69** | **95%** |

- 训练时间: 12 min
- DDPM 在 95% 样本上优于 LS，但平均 MAE 落后 UNet 0.84 rad

## 实验 2：DDIM 采样加速

### 改动
在 DDPM 类中添加 `ddim_sample()` 方法，支持确定性跳步采样。

### DDIM 速度-质量扫描 (50步)
| 步数 | 时间/样本 | Masked MAE | 加速比 |
|:---:|:--------:|:----------:|:-----:|
| 200 (DDPM) | 1240 ms | 6.32 | 1× |
| 100 | 595 ms | 6.26 | 2.1× |
| **50** | **298 ms** | **6.26** | **4.2×** |
| 20 | 118 ms | 6.25 | 10.5× |
| 10 | 60 ms | 6.26 | 20.8× |
| 5 | 30 ms | 6.26 | 41.7× |

### 结论
- DDIM 在 5-200 步范围内质量几乎不变 (**99% 质量保持**)
- 相位解缠是确定性逆问题，天然适合 DDIM 确定性采样
- 采用 DDIM-50 作为默认推理方式

## 实验 3：残差预测 (无效 ❌)

### 改动
训练目标从 `unwrapped` 改为 `residual = unwrapped - coarse_unwrapped`，推理时 `pred = coarse + ddpm_sample(residual)`。

### 结果
| 方法 | Masked MAE | Gain over LS |
|------|:---:|:---:|
| DDPM 残差 | 8.32 | +0.002 |

### 失败原因
- 残差信号太弱 (范围 [-6.6, 3.9])，DDPM 噪声尺度相对过大
- 模型学到的最优策略是预测零均值 → `coarse + 0 ≈ coarse ≈ LS`
- 残差预测在扩散模型框架下需要更强的条件信号支撑

---

## 实验 4：条件增强 (无效 ❌)

### 改动
条件通道从 3 增至 7：新增 `coarse_unwrapped`, `wrapped_gradient_x`, `wrapped_gradient_y`, `mask`。

同时测试了 **余弦噪声调度** 和 **掩膜感知损失**。

### 结果
| 指标 | 基线 (3ch) | 增强 (7ch+cosine+mask) | 变化 |
|------|:---:|:---:|:---:|
| Masked MAE | 8.33 | 8.31 | -0.02 |
| Gradient MAE | 0.69 | 0.69 | 0.00 |
| Win Rate | 95% | 95% | 0% |

### 失败原因
- 训练损失大幅下降 (0.183→0.040) 但解缠质量不变
- LS coarse 与 unwrapped 高度相关，不提供额外信息
- 128×128 中心裁剪几乎全是有效像素，mask loss 无影响
- 当前瓶颈不在条件信息量，而在**数据量和分辨率**

## 实验 5：提高分辨率 128→256 (有效 ✅)

### 配置
| 参数 | 变更 |
|------|------|
| 图像尺寸 | 128 → **256** |
| Batch | 8 → **4** (显存控制) |
| AMP | 新增 **torch.cuda.amp** (fp16) |
| Attention | @16 → **@32** |
| 训练时间 | 12 min → **25 min** |

### 结果
| 指标 | 128×128 | **256×256** | 改善 |
|------|:---:|:---:|:---:|
| DDPM Masked MAE | 8.33 | **7.55** | **-9.4%** |
| DDPM Gradient MAE | 0.69 | **0.61** | -11.6% |
| UNet Masked MAE | 7.49 | 6.79 | -9.3% |
| DDPM Win Rate | 95% | **96%** | +1% |
| DDPM vs UNet 差距 | 0.84 | 0.76 | 缩小 |

### 结论
- 分辨率提升是**唯一产生实质性改善的单一改动** (+9.4%)
- 128×128 中心裁剪丢失 93.75% 像素，严重限制模型能力
- AMP + batch=4 使 256 训练在 25min 完成，显存可控

## 实验总结

### 有效手段
| 排名 | 手段 | 效果 | 
|:---:|------|:---:|
| 1 | 提高分辨率 128→256 | MAE -9.4% |
| 2 | DDIM-50 加速 | 推理 4× 加速，质量不变 |
| 3 | AMP + 降 batch | 显存可控，256 可用 |

### 无效手段
| 手段 | 原因 |
|------|------|
| 条件增强 (3→7ch) | 条件信息已充足 |
| 残差预测 | 扩散噪声淹没了弱残差信号 |
| 余弦噪声调度 | 相位任务不需要精细低噪区间 |
| 掩膜感知损失 | 128 中心裁剪几乎无无效像素 |

### 当前最佳模型
- **配置**: 256×256, 3ch 条件, 200 步线性噪声, DDIM-50 推理
- **性能**: Masked MAE 7.55, Win Rate 96%, 训练 25 min
- **vs UNet**: 仍落后 0.76 rad，但差距在缩小

### 待探索方向
- 增加训练数据量 (300 → 1000+)
- 延长训练轮数 (80 → 100+)
- 梯度一致性损失
- 模型规模扩展

## 实验 6：A1 Wide-DDPM — 通道容量消融 (无效 ❌)

### 配置
| 参数 | 变更 |
|------|------|
| base channels | 64 → **96** |
| 参数量 | 12.6M → **27.9M** (2.2×) |
| 训练时间 | 25 min → **42 min** |
| 其余 | 与基线完全相同 |

### 结果
| 指标 | 基线 (ch=64) | Wide (ch=96) | 变化 |
|------|:---:|:---:|:---:|
| Masked MAE | 7.55 | 7.55 | 0.00 |
| Masked RMSE | 8.16 | 8.16 | 0.00 |
| Wrapped MAE | 1.64 | 1.65 | +0.01 |
| Gradient MAE | 0.61 | 0.58 | -0.03 |
| Win Rate | 96% | 96% | 0% |

### 结论
- **通道容量不是瓶颈**：增加 2.2× 参数量未带来任何 Masked MAE 改善
- 验证了实验设计假设：当前 UNet 容量已充足
- Gradient MAE 微降 (-0.03) 但不显著
- 下一步应转向**条件融合方式**或**任务专用结构**，而非纯容量消融

## 实验 7：C1 MinSNR — 时间步重加权 (无效 ❌)

### 配置
| 参数 | 变更 |
|------|------|
| 损失函数 | ε-MSE → **Min-SNR γ=5 加权** |
| 训练时间 | 25 min → **26 min** |
| 其余 | 与基线完全相同 |

### 结果
| 指标 | 基线 | MinSNR | 变化 |
|------|:---:|:---:|:---:|
| Masked MAE | 7.55 | 7.54 | -0.01 |
| Masked RMSE | 8.16 | 8.16 | 0.00 |
| Gradient MAE | 0.58 | 0.57 | -0.01 |
| Win Rate | 96% | 96% | 0% |

### 结论
- **Min-SNR 时间步重加权对相位解缠任务无效**
- 训练损失更低 (val 0.060 vs 0.103) 但不转化为解缠质量提升
- 相位任务的时间步之间可能不存在显著的优化冲突
- 训练策略改进（MinSNR/EMA）已基本排除，应集中精力到**条件注入方式改造**
